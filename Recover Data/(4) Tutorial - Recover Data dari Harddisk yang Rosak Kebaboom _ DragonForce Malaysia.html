Hi Gais.. Ragnar here. So semalam cousin aku datang rumah dengan muka sendu mintak tolong recover data dia dekat external harddisk yg dh corrupted kebabom. Dia ckp byk data dalam tu yang dia perlu so dia mnta tolong sgt2 dapat kan blik file2 peribadi tu. Aku tak berani nak kata boleh success 100% but we can try kan. Sebab aku bukan expert dan takda tools forensic khas, so memang susah sikit kalau nak manual recover balik file dari HDD rosak. Tapi aku try jelah. Biasanya, aku guna dua tools dari Linux dan hanya memerlukan dua command sahaja. Dan tak semua nya boleh berjaya 100%. So jangan letak harapan tinggi

Setup:
-mesin docking utk cucuk harddisk rosak
-Linux OS dkt Virtual Box pn boleh (debian, ubuntu..ni je yg pernah aku try)
-External/internal harddisk yg lebih besar size dr file yg nak di recover
-USB port yang berfungsi (sangat penting!)



kalau bukak dekat Windows, mmg xkan detect harddisk tu. Mungkin bahasa harddisk tu dh xboleh dibaca oleh sistem Windows kot.



So, aku try bukak dekat Linux. Baru boleh nampak. Haa yg tu lah HDD cousin aku.



Ohh sebelum aku lupa, kalau pakai Virtual Box pastikan USB korang dah bridge utk di detect dekat Linux. Tekan button + tu tekan mana satu USB korang. Settle.



Aku nak tau apa nama HDD tersebut dekat Linux aku so aku buat command
sudo fdisk -l
Yg last sekali tu L ya atau list. Macam yg korang nampak bawah sekali adalah external HDD dengan nama /dev/sdc1



Ok ni recovery command yang pertama.
sudo testdisk /dev/sdx
Yg mana x adalah nama HDD korang. Dlm case aku, /dev/sdc1.
Tools yg pertama ni memang dah built-in dalam Kali Linux so korang xperlu download lagi.



Dia akn keluar macam ni. Tunjuk HDD apa yg nak direcover. Tekan enter utk Proceed.



Just tekan enter apa yg dia Highlight (tengok Hint) biasa nya akan pergi Intel kt atas, tapi mungkin sebab HDD cousin aku ni dah kebaboom so dia tunjuk None. So just tekan enter.



Gerak arrow pergi kanan supaya highlight Image Creation, yang ni option utk buat backup image (DD file) of all file daripad corrupted HDD tadi. Tekan enter. So korang akan disuruh pilih partition mana korang nak simpan DD file tu. Just pilih mana tempat korang nak simpan. In real situation, kalau file asal corrupted HDD tu besar korang kena simpan DD file tu dekat another external HDD so locate Disk korang simpan DD File tu dekat sana. Utk DEMO ni, aku just save DD file tu dekat location /root/



So proses pengimejan DD file akan bermula. Selalu nya akan mengambil masa yang sangat lama bergantung kepada size asal corrupted HDD. Dalam kes aku, corrupted HDD ni 1TB.. So sempat lagi nak layan Series My Name dekat Netflix sampai habis.



So make sure ada DD file dekat lokasi korang letak tadi. Try tekan dia akan nampak peningkatan size makna nya pengimejan korang tengah working. Korang kena tunggu sampai 100% completed ya.



So, apabila DD file dh complete next step adalah command recovery kedua iaitu
sudo photorec /image.dd location/
Dalam kes aku sudo photorec image.dd sahaja sebab aku sekarang dekat directory root dan aku simpan image.dd tu dekat root aku. Lokasi image.dd tu penting!



Kalau command korang betul, dia akan tunjuk image.dd korang. just tekan enter to Proceed



Korang akan sampai dekat Interface ni. Sebelum korang tekan Search utk Start File Recovery. Korang boleh pergi ke File Option utk pilih file jenis apa yg korang nak recover utk jimat optimized kn masa recovery korang.



Gerak arrow ke kanan pergi File Opt. Tekan enter



Terdapat byk pilihan jenis2 file yg korang nak recover. Kalau aku, aku nak recover file doc sahaja so aku pilih doc sahaja. Tekan b utk save pilihan ini. Lepastu just Quit.



Then barulah korang tekan Search utk Start File Recovery



Dia akan tanya nak guna filesystem apa. Sekiranya Corrupt HDD tu asal nya OS Linux pilih ext2/ext3. Dalam kes aku, cousin guna OS Windows so guna Other. Tekan enter.



Pilih Whole. Pilihan ni lagi senang kerja. Tekan enter. Korang akan disuruh utk pilih directory kat mana korang nak extract data2 dr image.dd tu. Data2 ni adalah file2 asal & peribadi milik cousin aku yg ada kt dlm corrupted HDD tu. So kalau dalam real situation, korg kna pilih directory di external harddisk yg size lebih besar dari size corrupted HDD. Untuk demo ni, aku just extract data2 dr image.dd tu ke directory /root/ aku.



Apabila file berjaya diextract, file akan disimpan di dalam folder recup_dir di dalam directory yg korang set tadi.




Dalam folder2 recup_dir ni lah tersimpan data2 asal corrupted HDD tu... So klu korang bukak dlm folder tu akan tunjuk semua data2 asal dr corrupted HDD. Tapi naming dia xkan sama mcm yg asal. Contoh klu yg asal file nma dia bunga.xls, dia akan jadi kira2 mcm 110023478.xls. So korang kena hardworking sikit lah untuk cari balik dan naming semula semua file2 yg dh hilang tu. Asal dpt recover semua file dah cukup ber Syukur kan?

Ohh sebelum aku lupa, kalau pakai Virtual Box pastikan USB korang dah bridge utk di detect dekat Linux. Tekan button + tu tekan mana satu USB korang. Settle.


View attachment 14128
Aku nak tau apa nama HDD tersebut dekat Linux aku so aku buat command
sudo fdisk -l
Yg last sekali tu L ya atau list. Macam yg korang nampak bawah sekali adalah external HDD dengan nama /dev/sdc1


View attachment 14129
Ok ni recovery command yang pertama.
sudo testdisk /dev/sdx
Yg mana x adalah nama HDD korang. Dlm case aku, /dev/sdc1.
Tools yg pertama ni memang dah built-in dalam Kali Linux so korang xperlu download lagi.


View attachment 14132
Dia akn keluar macam ni. Tunjuk HDD apa yg nak direcover. Tekan enter utk Proceed.


View attachment 14131
Just tekan enter apa yg dia Highlight (tengok Hint) biasa nya akan pergi Intel kt atas, tapi mungkin sebab HDD cousin aku ni dah kebaboom so dia tunjuk None. So just tekan enter.


View attachment 14134
Gerak arrow pergi kanan supaya highlight Image Creation, yang ni option utk buat backup image (DD file) of all file daripad corrupted HDD tadi. Tekan enter. So korang akan disuruh pilih partition mana korang nak simpan DD file tu. Just pilih mana tempat korang nak simpan. In real situation, kalau file asal corrupted HDD tu besar korang kena simpan DD file tu dekat another external HDD so locate Disk korang simpan DD File tu dekat sana. Utk DEMO ni, aku just save DD file tu dekat location /root/


View attachment 14137
So proses pengimejan DD file akan bermula. Selalu nya akan mengambil masa yang sangat lama bergantung kepada size asal corrupted HDD. Dalam kes aku, corrupted HDD ni 1TB.. So sempat lagi nak layan Series My Name dekat Netflix sampai habis.


View attachment 14140
So make sure ada DD file dekat lokasi korang letak tadi. Try tekan dia akan nampak peningkatan size makna nya pengimejan korang tengah working. Korang kena tunggu sampai 100% completed ya.


View attachment 14143
So, apabila DD file dh complete next step adalah command recovery kedua iaitu
sudo photorec /image.dd location/
Dalam kes aku sudo photorec image.dd sahaja sebab aku sekarang dekat directory root dan aku simpan image.dd tu dekat root aku. Lokasi image.dd tu penting!


View attachment 14144
Kalau command korang betul, dia akan tunjuk image.dd korang. just tekan enter to Proceed


View attachment 14147
Korang akan sampai dekat Interface ni. Sebelum korang tekan Search utk Start File Recovery. Korang boleh pergi ke File Option utk pilih file jenis apa yg korang nak recover utk jimat optimized kn masa recovery korang.


View attachment 14154
Gerak arrow ke kanan pergi File Opt. Tekan enter


View attachment 14155
Terdapat byk pilihan jenis2 file yg korang nak recover. Kalau aku, aku nak recover file doc sahaja so aku pilih doc sahaja. Tekan b utk save pilihan ini. Lepastu just Quit.


View attachment 14156
Then barulah korang tekan Search utk Start File Recovery


View attachment 14149
Dia akan tanya nak guna filesystem apa. Sekiranya Corrupt HDD tu asal nya OS Linux pilih ext2/ext3. Dalam kes aku, cousin guna OS Windows so guna Other. Tekan enter.


View attachment 14150
Pilih Whole. Pilihan ni lagi senang kerja. Tekan enter. Korang akan disuruh utk pilih directory kat mana korang nak extract data2 dr image.dd tu. Data2 ni adalah file2 asal & peribadi milik cousin aku yg ada kt dlm corrupted HDD tu. So kalau dalam real situation, korg kna pilih directory di external harddisk yg size lebih besar dari size corrupted HDD. Untuk demo ni, aku just extract data2 dr image.dd tu ke directory /root/ aku.


View attachment 14152
Apabila file berjaya diextract, file akan disimpan di dalam folder recup_dir di dalam directory yg korang set tadi.


View attachment 14153

Dalam folder2 recup_dir ni lah tersimpan data2 asal corrupted HDD tu... So klu korang bukak dlm folder tu akan tunjuk semua data2 asal dr corrupted HDD. Tapi naming dia xkan sama mcm yg asal. Contoh klu yg asal file nma dia bunga.xls, dia akan jadi kira2 mcm 110023478.xls. So korang kena hardworking sikit lah untuk cari balik dan naming semula semua file2 yg dh hilang tu. Asal dpt recover semua file dah cukup ber Syukur kan?
